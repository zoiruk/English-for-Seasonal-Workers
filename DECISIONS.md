## Решения по сборке (зафиксировано 2026-06-16)

Владелец делает проект сначала для себя (осваивает английский, ~1 урок/день, полное
освоение урока перед переходом к следующему), затем — как стартап с продвижением через
TikTok. Планка качества высокая: каждый урок практичный, интерактивный, со звуком.

Старый общий курс «English from Zero» **переписываем под фермерский формат**. Принятый план:

- **Архитектура:** урок = единый объект `{grammar, words, dialogue, quiz}`; навигация
  Hub-and-Spoke (главная-хаб со списком 15 уроков → экран урока с табами
  Grammar/Words/Dialogue/Quiz). `app/` переписываем с нуля.
- **Данные:** один внешний `app/data.js` → `window.LESSONS = [ … ]` (без `fetch`, offline).
- **Старое в архиве:** `lessons/`, `vocabulary/`, `tracks/`, старые `README/PROGRESS/HOW-TO-STUDY`
  перенесены в `_legacy/` (не удалять — могут пригодиться как черновик грамматики).
- **Сначала лексикон, потом код (Фаза 0):** мастер-лексикон уроков 1–15 (en + IPA из словаря +
  ru + emoji) со сверкой по CEFR-PDF и стартовым whitelist снежка — одобряется владельцем ДО кода.
- **Сборка вертикальным срезом:** затем движок + эталонный Урок 1 по «золотому стандарту»,
  владелец проходит его как первый пользователь, затем тираж уроков 2–15 (по одному в день).
- **MVP-движок (готов):** хаб+прогресс, урок с 4 табами, Web Speech, квиз-тренажёр, streak, PWA.
- **Полный объём A0-A1 (решено 2026-06-18, maximum scope):** по сверке с 4 эталонными
  Starter-курсами (`specs/coverage-matrix.md`) проект расширяется до полноценной платформы.
  Сверх 15 уроков грамматики добавляем: функциональный слой «Everyday English» + classroom-
  survival (алфавит/«не понимаю»), разговорник выживания, аудирование (listen-choose), чтение
  (gist), продукцию (tap-to-build), режим повторения (SRS), сертификат. Дорожная карта и
  порядок сборки — `plans/2026-06-18-full-a0a1-scope.md`. Грамматика покрыта 1:1 с CEFR;
  главный пробел был — функциональный слой и перекос в recognition.
- **Git:** репозиторий инициализирован для обратимости (данные урока живут в JS).

## Reading — «Книга про Ахмада» (graded reader, 2026-06-22)

Согласовано гриллом (`plans/2026-06-22-reading-book-feature.md`). Почему так:

- **Зачем чтение вообще:** курс силён в recognition/диалоге, но связного экстенсивного чтения не было.
  Graded reader даёт сцепку уже выученного в нарратив + раннюю практику «читать латиницу» (цель проекта),
  без нового грамматического долга — только «to be» + this/that + притяжательные.
- **Статика, НЕ рантайм-генерация:** офлайн, нет LLM на устройстве, непроверенный английский = риск №1.
  Тексты пишутся заранее и проходят те же 4 слоя (вкл. source-cited натуральность), что и уроки.
- **Снежок + дозированный glossary:** тело главы N связано снежком (слова 1..N + whitelist); «новые» слова —
  осознанная добавка ≤5/главу, каждое в `glossary`, подсвечено синим, по тапу — попап. `reader-audit.js` ловит
  ЛЮБОЕ слово вне лексикон(1..N)+whitelist+glossary → нет молчаливого непроверенного английского.
- **Синие слова — только для чтения (реш. #10):** НЕ входят в `store.words`/SRS/мастер-лексикон. Иначе (а) раздувался
  бы счётчик «слов изучено» пассивным узнаванием, (б) конфликт дедупа, если слово позже станет словом урока.
- **Опоры (затухающие):** английский крупный и главный; кириллической транскрипции под ЦЕЛЫМИ фразами НЕТ
  (иначе читают кириллицу, не латиницу — против цели). Транскрипция живёт только в попапе синего слова.
  RU-перевод фразы — по тапу (скрыт по умолчанию).
- **Понимание — RU-вопросы:** 2–3 вопроса в стиле `[GIST]`, всё по-русски → активное чтение + сигнал «глава пройдена»,
  ноль нового непроверенного английского в проверке.
- **Разблок по `store.done[N]`:** глава N открывается после прохождения урока N — совпадает со снежком (глава N = слова ≤N).
- **v1 = пруф 3 глав, НЕ 15 — осознанный override:** приоритет №1 (живой A0-тест Б) не проведён, L11–L15 уже построены
  вперёд валидации. Сначала проверяем саму идею чтения на реальном пользователе ЦА, потом дописываем 4–15. Зафиксировано
  честно (как override-статус L11–L15).
