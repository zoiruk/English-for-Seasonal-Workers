**Дата:** 2026-06-22 · **Статус:** ✅ РЕАЛИЗОВАНО 2026-06-22 (пруф 3 глав L1–L3). План согласован гриллом.

**Решено гриллом с владельцем** (`/grill-me`). Все 10 веток дерева решений закрыты ниже.

## Что строим

Новый режим **Reading** в приложении: сквозная история про Ахмада (героя из L1),
разбитая на главы = уроки. Графированный ридер уровня A0–A1 под контекст сезонного
рабочего. Глава N опирается на лексику уроков 1..N; недостающие слова добавляются
дозировано, подсвечиваются синим, по тапу — попап с транскрипцией + переводом + 🔊.

## Согласованные решения (грилл)

1. **Генерация — НЕ в рантайме.** Тексты пишу заранее, верифицирую, зашиваю как
   `window.READER` (статичные данные, как `LESSONS`/`PHRASEBOOK`). Никакого LLM на
   устройстве — офлайн, риск №1 (непроверенный английский).
2. **Форма** — одна сквозная история про Ахмада, главы по урокам (приезд → люди →
   вещи на ферме → … → зарплата → врач → столовая). ~80–120 слов на главу.
3. **Лексика** — снежок + лимит **новых ≤5/глава** + явный `glossary`. Тело главы N —
   только слова из уроков 1..N + служебный whitelist (`scripts/lib.js`
   `activeWhitelist`). Новые слова — осознанная добавка, каждое в `glossary` главы с
   IPA+транскрипцией, сверено. **Скрипт-аудит ловит любое слово вне
   лексикон(1..N)+whitelist+glossary** → нет молчаливого непроверенного английского.
4. **Опоры (режим чтения)** — английский текст главный и крупный; 🔊 озвучка
   предложения; **RU-перевод открывается по тапу** на предложении. Кириллической
   транскрипции под целыми фразами НЕТ (иначе читают кириллицу, не латиницу — против
   цели проекта «учить читать латиницу, транскрипция — затухающая опора»).
   Транскрипция — только в попапе синего слова.
5. **Тап в тексте** — реагируют **только синие новые слова** → попап (слово +
   транскрипция + RU + 🔊 озвучка этого слова). Известные слова озвучиваются через 🔊
   всего предложения (одиночный Web-Speech по слову звучит хуже + легко промахнуться).
6. **Понимание** — в конце главы **2–3 лёгких RU-вопроса** по сюжету (стиль `[GIST]`:
   RU-вопрос, RU-варианты). Активное чтение + сигнал «глава пройдена». Ноль нового
   непроверенного английского (всё по-русски).
7. **Разблок** — глава N открывается **после прохождения урока N** (`store.done[N]`).
   Закрытые главы — серые с 🔒. Совпадает со снежком (глава N = слова ≤N).
8. **Объём v1** — **пруф из 3 глав + движок**, НЕ все 15. Причина: приоритет №1 по
   STATE.md — живой A0-тест Б, он не проведён; L11–L15 + ретрофит уже построены
   вперёд валидации. Сначала проверяем саму идею чтения на реальном пользователе,
   потом дописываем. **Это осознанный override владельца, как L11–L15 — зафиксировать
   в STATE.md честно.**
9. **3 главы пруфа** — подряд **L1–L2–L3** (знакомство → люди → вещи на ферме). Связно
   читается и это главы, до которых новичок ЦА доходит первыми (ранний разблок) →
   честный тест идеи.
10. **Синие слова — только для чтения.** НЕ попадают в `store.words`/SRS/мастер-
    лексикон уроков. Пассивное узнавание внутри книги. Избегаем конфликта дедупа
    (слово может позже стать словом урока) и раздувания счётчика «слов изучено».

## Формат данных (`app/reader.js` → `window.READER`)

```js
window.READER = [
  {
    id: 1,                         // == lesson id (гейт разблока store.done[id])
    emoji: "✈️",
    title_ru: "Ахмад приезжает",   // RU-заголовок главы
    title_en: "Ahmad arrives",     // опц., мелко
    sentences: [
      { en: "Hello! I am Ahmad.", ru: "Привет! Я — Ахмад." },
      // ... только слова уроков 1..N + whitelist + glossary[].en
    ],
    glossary: [                    // ≤5; рендер красит эти токены синим в тексте
      { en: "England", transcr: "инглэнд", ru: "Англия", pn: "/ˈɪŋɡlənd/" }
    ],
    quiz: [                        // 2–3 RU-вопроса (без [TAG], всё по-русски)
      { q_ru: "Как зовут героя?", opts_ru: ["Ахмад","Том","Али","Анна"], c: 0 }
    ]
  }
]
```

- **glossary v1 — одиночные слова** (не фразы), чтобы подсветка/матч были простыми.
  Если новая единица — фраза, глоссировать ключевое слово, остальное — из снежка.
- Транскрипцию глоссария писать вручную по конвенциям (`/w/`→«у», `æ`→«э», топонимы по
  рус. написанию) — `verify-transcr` покрывает только `words[]`, не reader.

## Движок (`app/app.js`)

- Хаб: карточка `data-nav="reader"` (📖 «Книга про Ахмада», прогресс «N глав открыто»).
  Класть рядом с phrasebook/review/cert.
- Роутинг: `#reader` (список глав) и `#reader/<id>` (глава). Закрытые id → редирект/🔒.
- `renderReaderList()` — главы списком; `store.done[id]` ? открыта : 🔒 серая.
- `renderChapter(ch)`:
  - предложения: каждое = строка с 🔊 (играет `en`) + текст `en`; тап на строке
    раскрывает `ru` (toggle, как «показать перевод»).
  - подсветка синего: для каждого предложения токенизировать, матчить токены против
    `ch.glossary[].en` (lowercase, базовая нормализация); матч → `<span class="g-new"
    data-gloss="en">`. Тап → попап (слово, транскрипция, ru, 🔊).
  - после предложений — 2–3 RU-вопроса (переиспользовать паттерн опций из квиза, но
    проще: RU q/opts, перемешивание опций как в `runQuiz`, `c` = истинный индекс).
  - завершение: `store.read = store.read || {}; store.read[id] = true; save()`.
    Стрик — бампать как у уроков? **Решить при реализации** (склоняюсь: да, чтение —
    учебное действие; но НЕ трогать `store.words`). 
- Озвучка — существующий `speak()` (en-GB, rate 0.88, `cancel()` перед).

## Верификация (обязательна перед коммитом каждой главы — слой-4 = риск №1)

1. **Новый скрипт `scripts/reader-audit.js`** (вне `npm run audit` основного, но
   добавить в pipeline): для каждой главы N — токенайз `sentences[].en`, проверить, что
   каждый стем ∈ {лексикон уроков 1..N} ∪ `activeWhitelist(N)` ∪ `glossary[].en`.
   Ошибка на любое непокрытое слово. Плюс проверки: `glossary.length ≤ 5`,
   у каждого glossary есть `transcr`+`pn`+`ru`, у quiz `c` в диапазоне и 4 опции.
2. **Транскрипция глоссария** — сверить вручную по конвенциям (+ при желании прогнать
   через логику `verify-transcr` точечно).
3. **🔴 Слой-4 натуральности** — `/lesson-naturalness-check` адаптированно: сверить
   английский глав против 5 эталонных курсов (graded-reader пассажи: Headway/Speakout/
   face2face/EF Beginner readers) + цитируемые ELT-источники. Source-cited, второй
   проход против ложных флагов. **Пропуск = глава НЕ готова.**
4. **Адверсариальное ревью** (язык/IPA + педагогика/контракт) — ловит фонетику/дрейф
   транскрипции в предложениях (скрипты этого не видят).
5. Бампнуть `CACHE_VERSION` в `app/sw.js` (сейчас `esw-v51`); добавить `reader.js` в
   `PRECACHE`; обновить `STATE.md` + `DECISIONS.md` (почему пруф, override-статус).
6. Подключить `reader.js` в `app/index.html` перед `app.js`.
