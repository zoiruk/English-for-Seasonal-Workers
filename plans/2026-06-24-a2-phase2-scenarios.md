## Сценарии (4 подтверждены + 1 TBD)
1. **Мне не заплатили / ошибка в зарплате** (`pay`) — разговор с менеджером/агентством; `haven't been paid`,
   payslip, wage/tax/account. Safety: ACAS/gov.uk про pay; нейтральные формулировки, без «вам должны / это незаконно».
2. **У врача / вызов скорой (NHS)** (`nhs`) — симптомы, 999 vs 111, регистрация у GP. Safety: NHS (nhs.uk) —
   когда 999/111; без диагнозов и мед.-советов, только «как сказать / куда звонить».
3. **Разговор с агентством/работодателем** (`agency`) — смена, часы, жильё, проблема на кэмпе.
4. **Авария на технике / травма на поле** (`accident`) — остановить работу, сообщить о поломке/травме,
   попросить помощь. Safety: HSE-принципы (остановись, сообщи), без инструкций по ремонту/первой помощи.
5. **«Something else»** — владелец отметил без деталей. **🔴 Открытый вопрос: уточнить тему перед сборкой
   этого сценария** (остальные 4 можно строить, не дожидаясь).

Старт: построить 3–4, расширять. Рекомендую начать с `pay` (самый частый, безопасная зона) как эталон
формата, затем `agency`, `accident`, `nhs` (строжайшая safety — последней, отдельным проходом).

## Формат данных — `app/scenarios.js` (`window.SCENARIOS`)
Как разговорник/ридер: отдельный файл, **вне пер-урочного снежка**, подключить в `index.html` перед `app.js`
и в `sw.js` PRECACHE. Модель — граф узлов «реплика собеседника → 2–4 варианта ответа игрока → следующий узел»:

```js
window.SCENARIOS = [{
  id: "pay", icon: "💷",
  title_ru: "Мне не заплатили",
  sub_ru: "Разговор с менеджером о зарплате",
  intro_ru: "Прошла неделя, но денег на счёте нет. Вы идёте к менеджеру.",
  start: "n1",
  nodes: {
    n1: { speaker: "m", en: "Hello, how can I help you?", transcr: "...", tr_ru: "...",
      choices: [
        { en: "I have not been paid this week.", transcr: "...", tr_ru: "...",
          ok: true,  fb_ru: "✅ Чётко и вежливо — так и нужно.", next: "n2" },
        { en: "Where is my money?", transcr: "...", tr_ru: "...",
          ok: false, fb_ru: "⚠️ Слишком резко. Лучше спокойно: «I have not been paid…»", next: "n1" }
      ] },
    n2: { speaker: "m", en: "Let me check your payslip.", ..., choices: [ ... ] },
    end_good: { speaker: "m", en: "I will check it today. Sorry about that.",
      transcr: "...", tr_ru: "...", outcome: "good",
      outcome_ru: "Вы спокойно объяснили проблему — менеджер проверит payslip. 👍" }
  }
}];
```
- `speaker`: `m`(manager)/`w`(worker)/`c`(cashier)/`d`(doctor)/`a`(agency)/`n`(nurse) — расширить набор в движке.
- Каждая `en`-строка (узлы + варианты) — с `transcr` (кириллица) и `tr_ru`. 5–15 слов.
- Узел-исход: `outcome:"good"|"ok"|"bad"` + `outcome_ru`. ≥2 узла-исхода (хороший/слабый), чтобы выбор имел вес.
- Варианты: ≥2, ровно один помечен «лучшим» путём; «плохой» вариант не тупик — даёт `fb_ru` и возвращает/ведёт
  к восстановительному узлу (учим, а не наказываем — как «Разбор ошибок» в квизе).

## Лексика — снежок-чистота против ВСЕГО лексикона A0–A2
Реплики = продакшн-английский, который рабочий реально скажет → должны быть verified. Но сценарии не привязаны
к уроку N: разрешена лексика **объединения всех уроков 1–23** (`words[]`) **+ стартовый whitelist + NAMES**.
Новые слова не вводить (это режим применения, не обучения). Грабли с сокращениями те же
([[contractions-break-snowball-tokenizer]]): токенайзер снимает `'ve/'re/'ll/'m/'d/n't` → `haven't`/`I've`
матчатся; проверить на конкретных формах.

## Новый аудит — `scripts/scenario-audit.js` (добавить в `npm run audit`)
Аналог `snowball.js`, но цель = **union всех `words[]` уроков 1–23** (+ whitelist + NAMES), не «1..N»:
- Лексика: каждый токен всех `en` (узлы + варианты) ∈ union-лексикон. Незнакомое слово = ошибка.
- Структура: `start` существует; каждый `next` указывает на существующий узел; нет недостижимых узлов;
  у не-исходных узлов `choices.length ≥ 2`; ≥1 узел с `outcome`; `transcr`+`tr_ru` есть у всех `en`.
- Транскрипция: прогнать сценарные `en` через `check-transcr-safe`/`check-transcr-consistency` (знакомые слова
  ≥5 букв из `words[]`/глоссария — единая транскрипция; кейс L7 `tomorrow`).
- BrE: `check-bre` (никаких «Good job»→«Well done», американизмов) [[bre-praise-good-job-vs-well-done]].

## Движок — `app/app.js`
- `renderScenarioList()` — хаб-вход «🆘 Ситуации» (плитка, как Разговорник 📒): список сценариев (icon, title_ru,
  sub_ru). Появляется, когда `SCENARIOS.length`.
- `renderScenario(id)` — экран: `intro_ru` → играем граф. Текущий узел: реплика собеседника (англ. + 🔊 + transcr +
  tr_ru) → кнопки-варианты (англ. + transcr + tr_ru). Тап → показать `fb_ru` (✅/⚠️), кнопка «Дальше» → `next`.
  Узел-исход: `outcome_ru` + «Пройти снова» / «К списку» / «На главную». Reuse `speak()` (Web Speech, en-GB, 0.88).
- Роутинг: `#scenarios` (список) и `#scenario/<id>` (как `#book/<id>`); `setRoute`; обработчик `data-nav="scenarios"`.
- A2-акцент допустим (фиолетовый), но это уровень-нейтральный режим — решить на месте (по умолчанию синий акцент).

## 🔴 4-слойный гейт (обязателен перед коммитом — это зона повышенной ответственности)
1. `npm run audit` (с новым `scenario-audit.js`) — зелёный.
2. Чек-лист CLAUDE.md: фермерский контекст, эмодзи по смыслу, RU-only, реальные `fb_ru`/`outcome_ru`.
3. Адверсариальные ревьюеры: (A) язык/IPA/BrE/transcr; (B) педагогика + **safety** (ни одной небезопасной
   денежной/мед./юр. строки; нейтральность; без claims).
4. **Source-cited safety + натуральность** (`/lesson-naturalness-check`, метод): каждая реплика с цитатой против
   5 ELT-источников + предметных (ACAS/gov.uk для pay, **nhs.uk** для NHS, HSE для аварии). Второй проход против
   ложных флагов. **Пропуск слоя 4 = не готово.**

## Кросс-каттинг / финал
