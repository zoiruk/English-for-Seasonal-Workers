## Контекст и решение владельца

Пакет 1 закрыл 6 блоков ядра (короткие гласные · magic e · TH/SH/CH/CK · EE/OO/EA ·
R-controlled · дифтонги AI/AY·OU/OW·OI/OY). Владелец прислал полный справочник фоники
(4 скана: Digraphs, IPA-таблица гласных/согласных, silent-letters, Combination of letters)
и указал, что ядро — лишь срез. **Решение владельца (2026-07-01, AskUserQuestion): вариант
«Ядро + IPA-справка».** То есть:
- добавить 3 блока ВЫСОКОЧАСТОТНЫХ комбинаций как интерактивные (drill), +
- добавить полную IPA-таблицу гласных/согласных (со скана стр.7) как СПРАВОЧНЫЙ экран без
  теста,
- РЕДКИЕ академические комбинации НЕ добавлять (rh/mn/ch=/k/·/ʃ//gh=/f/·/g//gn/tch/dg и
  дифтонги eə/ɪə/ʊə) — груз без отдачи для A0-аудитории (низкая грамотность, нулевой
  английский); они всё равно попадут в справочную IPA-таблицу «на просмотр».

## 🔴 Сканы = карта охвата, НЕ источник звуков (риск №1)

Присланные сканы содержат ОШИБКИ — не копировать дословно, каждый звук сверять с
Wiktionary-RP по одному слову (правило проекта после инцидента «three→Сри», commit `1f1838d`):
- **Стр.5 `all /ɔɪ/ ball` — прямая ошибка книги.** `ball` = `/ɔːl/` → «бол» (звук «о»), НЕ
  `/ɔɪ/` («ой» → «бойл» было бы грубо неверно). Блок ALL пишем через «о».
- **Стр.7 — американская нотация** (`/oʊ/ rose`, `/ɑː/`). Проект **BrE-strict** → `OA`
  (boat/goat) = `/əʊ/` → «оу» (как `home`→«хоум» в Пакете 1), НЕ US-«оа/оу».
- Небрежности нотации (`ea /ɪː/`, `oo /u/ book`) в модуле уже разведены правильно — не
  наследуем.

## Объём Пакета 2

### Блок 7 — Ещё согласные диграфы (высокочастотные)
`id: "consonant_digraphs_2"`, icon подобрать честный (напр. 🔔 для ng-звона? или 🅰️→нейтр.).
- `NG` → носовой «нг» (long/ring/sing)
- `PH` → «ф» (phone/photo)
- `WH` → «у» (what/white) + строка-исключение `WH→/h/` who «ху»
- `QU` → «ку» (/kw/, queen/quick)

### Блок 8 — Немые буквы
`id: "silent_letters"`, icon 🤫.
- `KN` → «н» (knife/knee, K не читается)
- `WR` → «р» (write/wrap, W не читается)
- `MB` → «м» (climb/thumb, B не читается)
- `LK` → «к» (walk/talk, L не читается)
- (GH-немое НЕ отдельной строкой — оно внутри IGH, см. блок 9, чтобы не дублировать
  night/high в двух блоках; в блоке 9 в правиле упомянуть «GH внутри не читается».)

### Блок 9 — Ещё гласные команды
`id: "vowel_teams_2"`, icon 🚣 (boat) или нейтр.
- `OA` → «оу» (/əʊ/, boat/goat/road)
- `UE` → «у» (/uː/, blue/true/glue)
- `IGH` → «ай» (/aɪ/, high/night/light; «GH тут молчит»)
- `ALL` → «ол» (/ɔːl/, ball/wall/call — НЕ «ойл»!)

### Справочный экран — полная IPA-таблица (без drill)
`id: "ipa_chart"`, `kind: "reference"` (новый тип блока). Со скана стр.7, BrE-исправлено:
- **Гласные:** долгие `iː`(sheep) `uː`(blue) `ɑː`(car) `ɔː`(door) `ɜː`(bird); короткие
  `ɪ`(fish) `e`(bed) `æ`(hat) `ʌ`(sun) `ɒ`(dog) `ʊ`(book); шва `ə`; дифтонги
  `eɪ`(cake) `aɪ`(night) `əʊ`(rose ← BrE, НЕ oʊ) `aʊ`(cow) `eə`(bear) `ɔɪ`(boy)
  `ɪə`(beer) `ʊə`(poor).
- **Согласные:** p b t d k g f v s z m n h l r w + `θ/ð`(th) `ʒ`(z) `ŋ`(ng) `ʃ`(sh)
  `tʃ`(ch) `dʒ`(j).
- Каждая строка: IPA-символ + ключевое слово (en + 🔊 + кир. transcr) + тип (RU: «долгий»/
  «короткий»/«дифтонг»/«шва»). Только просмотр + озвучка, теста нет.
- **Все ключевые слова, которых ещё нет в каноне, — через Фазу 0 (Wiktionary по одному).**

## Риски

- **Транскрипционный (риск №1)** — сканы с ошибками (`ball /ɔɪ/`, US-нотация). Каждое слово
  через Wiktionary-RP, не копировать книгу.
- **Reference-блок = новый тип рендера** — единственная правка движка; держать аддитивной,
  не ломать `patterns`+`check` и не трогать квиз-движок уроков.
- **Consistency-гейт может не видеть `ipa[]`** — проверить/добавить, иначе транскрипции
  таблицы пройдут мимо сверки.
- **Нефермерская лексика** (phone/photo/blue/queen) допустима (snowball-exempt), но без
  явного города/офиса; редкие академические слова (Chicago/rhetoric) исключены решением
  владельца.
