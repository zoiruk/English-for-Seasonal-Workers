## Цель и рамка решения

Отдельный **кросс-уровневый** модуль (A0→B1), который тренирует **ухо и образец**, а не
чтение. Декодинг (читать английское) уже закрыт системой `transcr`/IPA + алфавит L1 —
дублировать не будем. Незакрытый пробел и реальный риск на смене — **понять речь и быть
понятым**, поэтому ведём от слушания/разборчивости.

**Честная рамка (записать в UI):** офлайн нет надёжного STT → произношение пользователя
оценить НЕЛЬЗЯ. Модуль = (1) образец (TTS проговаривает, ученик повторяет, без оценки) +
(2) рецептивное различение на слух. «Поставим произношение» НЕ обещаем. TTS (en-GB) ≠ живой
акцент бригадира — это тоже подписать.

Архитектурно: standalone-модуль как `phrasebook.js`/`reader.js`/`scenarios.js` — **вне
снежка/аудита**, со своей сверкой. Аудируемый движок квизов (`app.js`) НЕ трогаем —
минимальные пары рисуем своим рендером.

## Объём (один пакет, ~6 блоков)

5 проблемных контрастов для СНГ-говорящих + 1 блок словесного ударения:

1. `th` — think / this (не «синк/фри/сри»)
2. `v` / `w` — very / well
3. финальное `-ed` — worked / cleaned
4. `h` / немое h — hand / (h)our
5. безударный шва — a / the / -er в конце (banana, supervisor)
6. **словесное ударение** — «где удар?» на знакомых многосложных фермерских словах

Каждый блок: ~6 слов-образцов (🔊 + повторить) + ~8 пунктов различения. Слова — **только из
уже введённых `words[]`** (тогда они уже прошли `verify-ipa`); новые — только с ручной
цитатой из Wiktionary.

## 🔴 Фаза 0 — Валидация ДО авторинга (гейт, не пропускать)

Главный риск модуля: **сам Web Speech (en-GB) может не различать контраст вслух**. Если TTS
произносит think и sink одинаково — drink-различение неотвечаемо и вредно. Это фонетический
аналог probe-validator из ридера.

- [~] **ОЖИДАЕТ ВЛАДЕЛЬЦА (нужны уши).** Probe-инструмент собран:
      `scratchpad/phonetics-probe.html` (en-GB, rate 0.88, cancel-before-speak — как в app).
      Владелец слушает каждую пару → отмечает «🟢 разные / 🔴 одинаковые» → присылает сводку.
      Агент выполнить не может: нет ушей + headless без TTS-голосов. Это один из немногих
      фонетических чеков, доступных нулевому-английскому владельцу (вопрос про звук, не про язык).
- [x] **Probe пройден владельцем (2026-06-28).** Голос: Microsoft Mark **en-US** (не en-GB —
      на машине владельца нет UK-голоса; для личного пользования это и есть его реальный звук;
      консонантные/ударные контрасты US=GB → вердикт переносится). **Вердикт:**
      - drill на различение: **TH (4/4), V/W (3/3), H (3/3)** — голос различает чисто;
      - drill на ударение: **ударение (4/4)**;
      - режим «образец» (drill снят): **шва** (по источнику) + **финал/-ed (#3, 2 из 4 неразличимы)**.
      Замечание: для BrE-фиделити доустановить UK-голос (Windows → Речь → English (UK)); не блокер.
- [x] **Source-cite контрастов выполнен** (2026-06-28). Все 6 — типичные ошибки рускоговорящих
      по UK/академ. источникам: [Pronunciation Studio](https://pronunciationstudio.com/russian-speakers-english-pronunciation-errors/)
      (th→s/z: think/bath; v/w: very/wild; финал. оглушение: bad/love; h→/x/ или немое: how/happy;
      шва: about/manner/sailor; ударение: любой слог), [British Accent Academy], [PMC L1-обзор].
      **Следствие:** шва — НЕ пара на различение, а «слышишь слабый слог» → режим «образец», не drill.

## Открытые вопросы / риски

- **Главный:** TTS-различимость контрастов (Фаза 0). Если 2–3 контраста не различимы — модуль
  усыхает до «образец+имитация+ударение». Это нормальный исход, не провал; решаем по факту.
- Минимальные пары требуют пар слов из известной лексики — где не хватает, берём блок целиком
  на «образец+имитация» без drill, а не тащим новые слова ради пары.
- Приоритет относительно отложенных B1-хвостов (разговорник/сертификат): этот модуль выше —
  даёт новую способность всем уровням, а не «ещё списки того же» (обосновано в обсуждении 2026-06-28).
