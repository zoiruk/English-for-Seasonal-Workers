# Валидация L1 — критерии pass/fail (зафиксированы ДО теста)

Контекст: совет (2026-06-20, уверенность high, 5/5) постановил остановить производство
контента и замкнуть петлю обратной связи. Это критерии для двух тестов. **Критерии
зафиксированы заранее, чтобы не подгонять вывод под желаемое.**

## Тест А — догфуд агентом (сегодня, без рекрутинга)
Эмуляция бюджетного Android: вьюпорт ~360px, чистка `localStorage` (первый запуск),
прохождение L1 как первый-раз-пользователь. **Ловит грубые блокеры доступности/онбординга,
НЕ обучение/удержание** (это только живой человек). Цель — стены RU-интерфейса и цифровой
грамотности.

| # | Критерий | PASS | FAIL |
|---|----------|------|------|
| C1 | Холодный старт | хаб рендерится за <3с на 360px, без горизонтального скролла, читабельно | белый экран / скролл вбок / нечитаемо |
| C2 | Аффорданс «как начать» | за ~10с видно, как открыть L1, по визуальным меткам (не требуя чтения абзаца по-русски) | непонятно куда жать без инструкции |
| C3 | Понятность кнопок (RU) | базовые действия (старт/далее/назад/табы) понятны при слабом русском | навигация требует B1-русского |
| C4 | Прохождение L1 | урок проходится от начала до конца (все табы + квиз) без тупика | тупик / непонятно как двигаться дальше |
| C5 | Аудио | кнопка 🔊/▶ очевидна и озвучивает | непонятно как слушать / молчит без сообщения |
| C6 | Квиз UX | как отвечать — очевидно; обратная связь верно/неверно понятна; опции перемешаны | непонятно как отвечать / ложный 100% |
| C7 | Сохранение прогресса | прогресс переживает обычную перезагрузку | сбрасывается на reload |
| C8 | Тач-таргеты | интерактив ≥44px (DESIGN.md/WCAG) | мелкие кнопки, мимо-промахи |

**Kill-condition теста А:** если первый-раз-пользователь не может из холодного старта
дойти до запуска L1 без посторонней помощи (C2 FAIL) ИЛИ не может пройти L1 (C4 FAIL) —
формат соло-самообучения сломан в основе → НЕ строить L13–L15, чинить онбординг/пересмотреть формат.

## Тест Б — живой A0-пользователь (до 27.06.2026, действие владельца)
Один реальный слабо-русскоязычный A0-взрослый из визового коридора, телефон, L1, наблюдение 20 мин.

| # | Критерий | PASS | FAIL |
|---|----------|------|------|
| L1 | Открыл сам за <2 мин без помощи | да | нет |
| L2 | Понял первую кнопку/действие | да | спросил «что нажимать» |
| L3 | Дошёл до конца L1 | да | бросил раньше |
| L4 | Что-то усвоил (5 слов спросить после) | ≥3/5 | <3/5 |
| L5 | Захотел вернуться завтра | да | нет |

**Kill-condition теста Б:** L1 или L3 FAIL → базовая гипотеза (A0-мигрант сам учится по
продукту) не подтверждена; дистрибуция/формат важнее нового контента.

### Протокол проведения теста Б (grill 2026-06-20, выбор владельца: вариант A)
- **Устройство:** ваш компьютер/телефон, лицом к лицу (первый сигнал; стены RU-интерфейса
  device-независимы). Телефон тестера по ссылке — следующий шаг, после деплоя.
- **Промпт:** одна нейтральная фраза («Вот, приложение учить английский — попробуй») и дальше
  **МОЛЧАНИЕ**. Не подсказывать, не вести за руку.
- **Наблюдение:** засечь время до старта L1; отметить КАЖДОЕ место заминки/вопроса «что нажать».
  Застрял >30с — это находка (записать дословно, что именно непонятно), НЕ спасать.
- **После урока:** спросить 5 английских слов из L1 (recall) и «вернёшься завтра?».
- **Что НЕ делать:** не объяснять интерфейс, не переводить кнопки, не защищать продукт. Цель —
  поймать реальность, а не доказать, что он хорош.

### Решающее правило (зафиксировано ДО теста)
- **PASS** (открыл сам <2 мин · понял первую кнопку · дошёл до конца L1 · ≥3/5 слов · хочет вернуться)
  → база работает → **деплой на GitHub Pages** (тест с реального телефона) + можно возвращаться к L13–L15.
- **FAIL** (бросил / не понял интерфейс / 0 удержания) → формат соло-самообучения под вопросом →
  чинить онбординг или пересмотреть формат/канал (фермы-агентства), **НЕ плодить новые уроки**.
