## Схема главы (из шапки `app/reader.js` — соблюдать точно)
```js
{ id,                              // == id урока (16..23); гейт открытия store.done[id]
  emoji, title_ru, title_en,
  sentences: [ { en, ru } ],        // тело: лексика уроков 1..N + whitelist + glossary[].en
  glossary:  [ { en, transcr, ru, pn } ],   // ≤5 НОВЫХ слов; в тексте подсвечены синим, попап
  quiz:      [ { q_ru, opts_ru:[4], c } ]   // 2-3 RU-вопроса на понимание, ВСЁ по-русски
}
```
- **`sentences`**: 12–16 предложений на главу (как в L1–L15). Каждый токен ∈ слова уроков 1..N (raw+stem)
  + активный whitelist + `glossary[].en`. Иначе `reader-audit` падает.
- **`glossary`**: ≤5, **одно слово** на элемент (`/\s/` запрещён), все 4 поля (`en/transcr/pn/ru`),
  и каждое слово ДОЛЖНО появиться в `sentences` (иначе ошибка «never appears»). Подаются как дозированные
  новые слова — синие, попап. **В `store.words`/SRS НЕ попадают** (пассивное узнавание).
- **`quiz`**: 2–3 вопроса, `opts_ru` ровно 4 (уникальные), `c` — индекс верного в диапазоне. Всё RU.
- **Транскрипция глоссария — вручную** по конвенциям приложения (`verify-transcr` НЕ покрывает ридер).
  **IPA `pn` — из Wiktionary/Cambridge (RP), не выдумывать** (это риск №1). `check-transcr-safe` и
  `check-transcr-consistency` СКАНИРУЮТ ридер-глоссарий → грубых подстрок нет, знакомые ≥5-букв слова — канон.

## Снежок ридера (КЛЮЧЕВОЕ отличие от сценариев/разговорника)
Ридер **снежок-СВЯЗАН**: глава N = слова уроков **1..N** (не union всех 23!). Глава 16 = слова L1–L16,
глава 20 = L1–L20 и т.д. Это нарастающая история. `scripts/reader-audit.js` проверяет токен-в-токен.
- Токенайзер `lib.js` стеммит и снимает `'s`; имена в `NAMES`. **Сокращения `'ve/'ll/n't` НЕ снимаются**
  → в `sentences` плейн/полные формы (как в уроках; грабля `contractions-break-snowball-tokenizer`).
- Короткие плюрали ≤3-букв стеммятся в кашу (`beds`→`b`) → избегать в снежок-полях.
- Глоссарий конкретной главы — НЕ слово какого-либо урока (это и есть «новое»); whitelist НЕ стеммится.

| Гл | Урок (тема) | Сюжет главы (черновик — уточнить при сборке) |
|----|-------------|----------------------------------------------|
| 16 | L16 be going to · погода | Ахмад вернулся на новый сезон; смотрит прогноз, планы на завтра (going to) |
| 17 | L17 сравнения | Новые рабочие; кто быстрее/сильнее; ящики тяжелее/легче (comparatives) |
| 18 | L18 could · техника | Учится водить трактор/работать аккуратно; что смог/не смог (could + adverbs) |
| 19 | L19 have got · тело/самочувствие | Устал, болит спина/плечо; «у меня есть…» (have got) |
| 20 | L20 something/anything/nothing · кэмп | Что-то/ничего в комнате/шкафу; обустройство (something/nothing) |

## Процесс на КАЖДУЮ главу (порядок)
1. Черновик `sentences` (12–16) под тему урока N, ТОЛЬКО лексика 1..N + whitelist + ≤5 глоссарий.
   - До написания — дамп доступной лексики: `node -e "..."` union слов уроков 1..N (как делали для сценариев),
     чтобы не ловить снежок-ошибки постфактум.
2. Глоссарий ≤5: IPA из Wiktionary (RP), transcr по конвенциям, каждое слово — в `sentences`.
3. Квиз 2–3 RU-вопроса (4 опции, верный `c`).
4. `npm run audit` → **зелёный** (`reader-audit` + `check-transcr-safe`/`-consistency` среди прочих).
5. **🔴 Слой-4 гейт натуральности** (`/lesson-naturalness-check` метод ИЛИ адверсариальный агент): сюжет +
   язык против graded-reader-практики 5 источников; второй проход против ложных флагов. Пропуск = НЕ готово.
6. После всех 8 (или пачками): бамп `CACHE_VERSION` в `app/sw.js`; обновить `STATE.md`; коммит по именам.

## 4-слойный гейт (как везде)
1. `npm run audit` зелёный (вкл. `reader-audit`).
2. Чек-лист `CLAUDE.md`: фермерский контекст, эмодзи по смыслу, RU-only, реальные RU-вопросы квиза.
3. Адверсариальные ревьюеры: (A) язык/IPA/BrE/transcr глоссария; (B) педагогика — сложность ровно A2,
   снежок честный (глоссарий ≠ слово урока), сюжетная связность с гл.1–15.
4. **Source-cited натуральность** против 5 источников (graded-reader регистр). Это неИИ-якорь.
