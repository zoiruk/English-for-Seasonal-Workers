# DESIGN.md — дизайн и доступность

UI-токены и правила доступности. Читается при работе над интерфейсом
(`app/style.css`, экраны, вёрстка). Дизайн-решение: **только светлая тема.**

## Дизайн (Telegram Web 2026, light-only)

- **Только светлая тема.** Тёмной нет — не добавлять.
- Clean Light: фон `#f0f2f5`, карточки `#fff`, текст `#1a1a2e`, акцент-синий `#0075de`.
- **Три уровня — три акцента.** Уровень 1 (A0–A1) — синий `--accent #0075de`. Уровень 2 (A2) —
  фиолетовый `--accent-a2 #6b46c1` (+ `--accent-a2-soft #f0ebfb`). Уровень 3 (B1) — бирюзовый
  `--accent-b1 #0f766e` (+ `--accent-b1-soft #e3f2ef`). Акцент применяется к карточкам урока
  (`.lesson-card.a2/.b1`: левый бордюр + бейдж-номер), к заголовку уровня (`.level-head.a2/.b1`), к шагу
  «Уровень N» в «пути» и к сертификату уровня (`.cert.a2/.b1 .cert-course` — фиолетовая/бирюзовая печать +
  цвет ленты печати-SVG). Контраст на белом ≈5–6.5:1 (WCAG AA). Цель — мгновенно «уровни, я расту».
- Системный sans-serif (`-apple-system, 'Segoe UI', Roboto`), без Google Fonts-зависимостей.
- Контейнер `max-width:430px; margin:0 auto` (мобильный приоритет, 320–430px).
- Навигация Hub-and-Spoke: главная-хаб + плавающая frosted-glass кнопка «‹ Назад» (iOS-chevron,
  `position:fixed`). Из хаба, кроме уроков, ведут плитки-входы: Разговорник 📒, Повторение 🔁
  (SRS, виден когда есть слова к повтору), Сертификат 🏆 — у каждого экрана своя frosted «Назад».
  Внутри урока — табы Grammar/Words/Dialogue/(Phrases)/Quiz; общая кнопка
  «Далее ➔» ведёт к следующему табу. В шапке урока — пилюля «N/всего».
- Карточки `border-radius:12–16px`; чипы pill `99px`; прогресс-бар `transition:width .5s`;
  нажатие `scale(0.98)`. Словарь — сетка 2 колонки (моб.) / 3 (широкий).
- **«Ваш путь» (`.journey`)** — карта-прогресс вверху хаба (показывается, когда есть A2-уроки):
  шаги «Уровень 1 / 2 / 3» (третий — когда есть B1-уроки) с per-level мини-баром, дробью `N/всего`, ✓ при завершении и
  🔒 если уровень закрыт; маркер `📍 вы здесь` на текущем уровне; соединитель `.jconn.on` зелёный,
  когда уровень 1 пройден. Всё на лёгком CSS (без тяжёлых анимаций — бюджетный Android).
- **Веха (`.milestone`)** — экран-празднование при завершении уровня A2 или B1 (последний урок уровня),
  показывается один раз на уровень (`store.msA2Shown` / `store.msB1Shown`). Заголовок берёт акцент уровня
  (`.milestone.b1 h1` — бирюзовый). Лёгкая анимация `ms-pop` (0.5s scale-in эмодзи), кнопки «Открыть
  сертификат A2/B1» / «На главную». Без heavy-эффектов.
- **Ситуации выживания (`.sc-*`)** — режим role-play (хаб-плитка 🆘, `data-nav="scenarios"`). Реплика
  собеседника — `.sc-bubble` (карточка с левым синим бордюром): `.sc-speaker` (метка роли, uppercase),
  кнопка 🔊, `.sc-en` (англ., акцент-синий), `.sc-transcr`, `.sc-ru`. Варианты ответа — `.sc-choices`
  (вертикальный стек, gap 10px) из `.sc-choice` (кнопка, touch-зона ≥44px, `.sc-c-en` + `.sc-c-tr`);
  после выбора все `disabled`. Фидбэк — `.sc-feedback.ok` (зелёный) / `.sc-feedback.warn` (оранжевый,
  токены `--hint-*`). Экран-исход — `.sc-outcome.good`/`.bad`/`.ok` + `.sc-actions` (кнопки «Пройти
  снова» / «К списку»). Акцент синий (уровень-нейтральный режим), без тяжёлых анимаций.

## Доступность

- Шрифт ≥14px (контент), ≥11px (метки/транскрипция); line-height ≥1.4; контраст WCAG AA.
- Touch target ≥44×44px, отступ между кнопками ≥8px, видимый отклик на нажатие.
- Не полагаться только на цвет: ✓ зелёный + «Правильно!», ✗ красный + «Неверно».
- Без горизонтального скролла: таблицы переносятся, кнопки 🔊 всегда в зоне видимости.

## Грабли превью

- **Скриншот превью таймаутит на `backdrop-filter`** (плавающая «Назад») — проверяй через
  `preview_eval`/computed-style, не скриншотами.
- **Кэш превью:** `python http.server` слал файлы без cache-заголовков → старое в браузере.
  Поэтому `npm run serve` = Node с `no-store`, а SW — `network-first`. После правок бампай
  `CACHE_VERSION`; в браузере Ctrl+Shift+R.
