# CLAUDE.md — English for Seasonal Workers

Контракт проекта (только инварианты-правила; журнал/история/UI вынесены — см.
«Карта файлов»). Подгружается в каждую сессию, **переопределяет** дефолтное поведение.
Язык с владельцем — русский; код/комментарии/коммиты — английский.

## Что это

Бесплатная веб-платформа изучения английского **с нуля (A0→A1)** для **сезонных
рабочих** из СНГ (Seasonal Worker Visa) в Великобритании.

- **Аудитория:** мигранты 20–45 лет, низкая цифровая грамотность, нулевой английский,
  бюджетный Android, нестабильный интернет, 15–30 мин вечером после смены.
- **Интерфейс: только русский (RU-only).** Мультиязычности (UZ/TJ/KG/KZ) сейчас НЕТ —
  не добавлять без явной просьбы (см. «Чего не делать»).
- **Контекст всего контента — ферма:** поля, пакхаусы, кэмпы, смены, безопасность,
  зарплата, документы. Слова/диалоги/примеры — только из жизни рабочего.

## Процесс работы (как делать урок)

1. Контент урока — в `app/data.js`; whitelist служебных слов и имена — в `scripts/lib.js`.
2. **`npm run audit`** (audit/dedup/snowball) — обязан быть зелёным перед коммитом.
3. **4-слойная проверка (все обязательны перед коммитом урока):**
   (1) автотесты `npm run audit`; (2) чек-лист по этому файлу (фермерская лексика, эмодзи
   по смыслу, RU-only, реальные `expl`); (3) **адверсариальные агенты-ревьюеры** (язык/IPA +
   педагогика/контракт) — ловят фонетику, чего скрипты не видят; (4) **🔴 source-cited сверка
   натуральности против 5 источников** (`/lesson-naturalness-check <Ln>`, метод `0ab5e1f`):
   каждый вердикт с цитатой + второй проход против ложных флагов. **Это неИИ-якорь против
   риска №1; пропуск слоя (4) = урок НЕ готов.** Слои 3–4 вне `npm run audit` (сеть/агенты).
4. **Коммит по именам файлов** после урока; бампнуть `CACHE_VERSION` в `app/sw.js`;
   **обновить `STATE.md`** (готовность урока + отчёты сверок).
5. Превью: **`npm run serve`** (Node-сервер с `no-store`) → `localhost:8000`. UI-правила → `DESIGN.md`.

## Золотой стандарт контента (педагогика — НЕ нарушать)

1. **Снежный ком:** в грамматике/диалогах/тестах урока N — ТОЛЬКО слова из словарей уроков
   1…N **плюс стартовый whitelist служебных/грамматических слов** (`am/is/are`, `not`, `do`,
   `a/the`, `this`, `where/what`, местоимения…), вводимых в grammar и не входящих в 30 тематических.
2. **Уникальность словаря:** одно англ. слово появляется в `words` только в одном уроке за курс.
3. **Транскрипция везде:** кириллическая `transcr` обязательна для каждого слова, примера,
   строки таблицы и реплики; в правилах англ. слова оборачивать `<span class="g-transcr">`.
4. **Нулевой уровень:** без лингвотерминов; базовые частицы/местоимения переводить явно (`I am (я)`).
5. **Только фермерская лексика:** запрещены городские/офисные/нерелевантные слова
   (printer, computer, apple…). Один урок = одна тема.
6. **Полнота:** grammar ≥10 примеров, words ≥30, dialogue ≥10, quiz ≥10. Меньше — урок не готов.

> Правила (1)(2)(6) проверяются `scripts/`: `audit.js` (структура), `dedup.js` (уникальность),
> `snowball.js` (снежок) → **`npm run audit`**. Пустой список ошибок → урок «золотой».

## Формат данных урока (RU-only)

Урок = объект `{grammar, words, dialogue, quiz}` в `app/data.js` → `window.LESSONS`. Поля
русские (`title_ru`, `rule_ru`, `tr_ru`, `hint_ru`, `ru`) + англ. `en`, IPA `pn`, кир. `transcr`.
**Localization-ready:** суффикс `_ru` намеренный — языки добавляются дописыванием `_uz/_tj/…`
рядом. UI-строки централизованно (`UI.ru = {…}` + хелпер `t(key)`).

- **grammar:** 3 формы (✅ positive / ❌ negative / ❓ question), у каждой `table`
  (`subj/verb/example/transcr/tr_ru`); `intro` через ❌/✅; `cultural` (отличие от русского);
  `note` (типичная ошибка ⚠️). Списки слов в правилах — CSS-grid, не строкой. Опц. помощь
  «ещё проще»: `simple_ru:{formula, examples:[{en,transcr,ru}]}` (≥1, снежок-чисты) + `ytQuery`
  (ручной RU-запрос) → свёрнутый блок в конце таба. Нет полей → блок не показывается.
- **words:** объекты `{e, en, transcr, ru, pn}`, по одному слову на элемент (запрещено `en:'one, two'`).
- **dialogue:** чередование спикеров `s`: `m`(manager)/`w`(worker)/`c`(cashier)/`d`(doctor);
  реплика 5–15 слов.
- **quiz:** задания `{q, opts[4], c, expl, hint_ru}`. Инструкции — только тегами
  `[COMPLETE]`/`[TRANSLATE]`/`[NEGATIVE]`/`[CORRECT]`/`[QUESTION]` + навыковые `[LISTEN]`
  (аудио ▶ → выбрать перевод, англ. скрыт), `[GIST]` (RU-вопрос по диалогу) и **продуктивный
  `[BUILD]`** (tap-to-build: вместо `opts/c` поле `build:[слова в верном порядке]` (≥2); движок
  перемешивает плитки → собрать фразу → проверка порядка; `expl` начинать с самой англ. фразы —
  движок показывает её в фидбэке; снежок проверяет слова `build`). Англ. инструкций
  в тексте быть не должно. `expl` обязателен. Аудит: 4 уникальные опции, индекс `c` в диапазоне,
  **answer-leak** (ответ не в вопросе, кроме `[TRANSLATE]`/`[LISTEN]`), уникальность реплик.
  **Движок перемешивает опции при каждом рендере (`app.js`), поэтому `c` = ИСТИННЫЙ индекс верного
  ответа — распределять по 0–3 не нужно (был баг: всё на `c:0` → урок проходился тыком первой кнопки,
  `97ff70c`). Неверный `c` теперь молча помечает неверную опцию.** [[quiz-options-shuffled-by-engine]]
- **everyday (опц.):** блок «Фразы» `{title_ru, phrases:[{en,transcr,ru}]}` (≥3), **вне снежка**;
  есть → у урока 5-й таб «Фразы». Опц. `html` — сырой HTML-справочник (grid) над фразами (алфавит L1,
  числа 11–100 L4); **вне снежка/аудита**, англ. внутри не проверяется — транскрипцию писать по конвенциям вручную.

## Грабли (на что уже наступали)

- **Зелёный аудит ≠ корректно, если whitelist неверен.** Был баг: `my/your` ошибочно в whitelist
  L1 → снежок проходил, но утечка темы L2. Всегда сверяй whitelist урока с его грамматикой.
- **Эмодзи-обманки** ревьюеры ловят часто. Нет точного эмодзи → бери ближайшее честное ИЛИ
  замени слово на фермерское с прямым эмодзи.
- **Кириллическая `transcr` — фактический ввод** (латиницу A0 не читает) → может сложиться в
  грубое слово. Кейс: `three`→«сри» → «фри» (`1f1838d`). Сверяй `npm run verify-transcr` (звук) +
  `verify-ipa` (IPA). Конвенции: `/w/`→«у», `æ`→«э», топонимы по рус. написанию. ⚠️ Оба скрипта
  покрывают ТОЛЬКО `words[]`, НЕ транскрипцию в примерах/таблицах/диалогах/`simple_ru` — там дрейф
  ловит лишь адверсариальное ревью (кейс L7 `tomorrow` «тумороу»→«тэмороу», `ced0a86`). Транскрипцию
  во фразах бери дословно из `words[]`.
- **Словоформы в снежке:** токенайзер в `lib.js` снимает `'s` и стеммит (`picking→pick`); имена в
  `NAMES`; `everyday` снежком НЕ проверяется. ⚠️ Снимает ТОЛЬКО `'s`, НЕ `'ve`/`n't`/`'ll`/`'m`/`'d`
  → `I've got`/`haven't`/`I'll`/`can't` падают в снежке. Решение L8 (грилл): снежок-поля — плейн/полные
  формы, сокращённые — только в `everyday`/`cultural`; токенайзер не трогать без согласия владельца.
  Повторится на Present Perfect/will/can't (L14). [[contractions-break-snowball-tokenizer]]
  ⚠️ **Короткие слова (≤3 букв) НЕ стеммятся**, но их плюрал стеммится в кашу (`beds`→`b`) → не матчит
  снежок (кейс L9). В снежок-полях избегай плюрала ≤3-буквенных: «some **chairs/blankets**», не «beds».
- **Объектные me/him/us** на A0 без глаголов не ввести → введены в **L6** с Present Simple (НЕ на L3).
- **Большие фан-ауты (18–30 агентов) в одном workflow выбивают лимит сессии** → меньший
  параллелизм / последовательные мелкие workflow. [[workflow-fanout-session-limit]]

## Контент-валидация и риски

- **IPA — из авторитетного словаря** (Wiktionary/CMUdict), не выдумывать. `transcr` — производная
  от IPA, подаётся как **затухающая опора** (больше в ранних уроках): цель — научить читать латиницу
  (payslip’ы, таблички). Слово всегда латиницей + 🔊; кириллица вторична.
- **Мастер-лексикон (`master-lexicon.md`)** — все слова с IPA, сверенные с CEFR-PDF; снимает разом
  снежок, дедуп, CEFR-объём и галлюцинации IPA.
- **Зона повышенной ответственности:** деньги/зарплата, права, NHS/медицина, безопасность — «лексика
  выживания», неверная фраза = реальный вред. Проверять строже, без юр./мед. утверждений; табу про
  визу/заработок сохраняется.
- **CEFR — это области лексики, не потолок слов.** Фермерская лексика = фермерское подмножество
  CEFR-областей (jobs→picker/packer, objects→punnet/crate).
- **Web Speech офлайн-зависим:** нет голоса/сети → не молчать, показать сообщение и текст.
- **localStorage хрупок** (чистка кэша стирает прогресс) → позже экспорт/импорт.
- **Владелец ≠ репрезентативный пользователь** (грамотнее/мотивированнее аудитории): проверяет
  педагогику, но не доступность для A0/низкой цифровой грамотности.

## Архитектура (Zero-Budget, Offline-First)

Без сервера, БД, внешних зависимостей (никаких JS/CSS-фреймворков).

- **Стек:** HTML + CSS + vanilla JS. Хостинг — GitHub Pages: деплой `app/` через
  `.github/workflows/pages.yml` (нужен ПУБЛИЧНЫЙ репо + Settings→Pages→Source «GitHub Actions»;
  пути относительные → работает на под-пути `user.github.io/<repo>/`). Repo/remote пока не создан.
- **Аудио:** Web Speech API, `lang='en-GB'`, `rate=0.88`; всегда `speechSynthesis.cancel()` перед
  воспроизведением. Нет интернета → остановить и показать сообщение.
- **Хранение:** localStorage (прогресс, счётчик слов, streak, сертификат). Сервера/аутентификации нет.
- **Иконки:** emoji строго по смыслу слова; картинок-файлов нет.
- **Производительность:** < 3 c на 3G, данные урока inline, offline после первой загрузки.
- **Совместимость:** Chrome/Samsung Internet/Safari iOS; Firefox Android — частично (Speech API).
- **Дизайн/доступность** (цвета, шрифты, layout, WCAG, touch-target) → `DESIGN.md`.

## Git-безопасность

Данные уроков живут в JS → откат через git может стереть невосстановимый контент. Частые коммиты
после каждого урока; бэкап массива перед опасными операциями; стадить по имени, не `git add .`.
«Откати» = неразрушающий просмотр (`git checkout <commit>`/`stash`/`log`), НЕ `reset --hard`.
⚠️ Автономный `/goal` с `git reset --hard` к контент-файлам не применять без явного бэкапа.
