# CLAUDE.md — English for Seasonal Workers

Контракт проекта. Подгружается в каждую сессию. Правила здесь **переопределяют**
дефолтное поведение. Язык общения с владельцем — русский, код/комментарии/коммиты —
английский.

## Что это

Бесплатная веб-платформа изучения английского **с нуля (A0→A1)** для **сезонных
рабочих** из СНГ (Seasonal Worker Visa) в Великобритании. Цель — помочь выжить и
адаптироваться с первого дня на ферме.

- **Аудитория:** мигранты 20–45 лет, низкая цифровая грамотность, нулевой английский,
  бюджетный Android, нестабильный интернет, 15–30 мин вечером после смены.
- **Интерфейс: только русский (RU-only).** Мультиязычности (UZ/TJ/KG/KZ) сейчас НЕТ —
  не добавлять без явной просьбы. См. [decision](#чего-не-делать).
- **Контекст всего контента — ферма:** поля, пакхаусы, кэмпы, смены, безопасность,
  зарплата, документы. Слова/диалоги/примеры — только из жизни рабочего.

## Источник правды

1. **CEFR-PDF в `specs/`** (`CEFR A0-A1 … C1 …outline.pdf`) — канонический объём
   грамматики и лексики по уровням. Строить уроки строго в их рамках. MVP = A0-A1.
2. **`specs/*.md`** — продуктовые/тех/контент-спеки. ВАЖНО: они пришли из родственного
   проекта и описывают мультиязычную версию (`a1.html`, `translations.js`, `scratch/`,
   ключи `fe_*`, 5 языков). **Эти пути и 5-язычность к нам не относятся** — брать из них
   только принципы, адаптируя под RU-only и под реальную структуру (ниже).

## Решения по сборке (зафиксировано 2026-06-16)

Владелец делает проект сначала для себя (осваивает английский, ~1 урок/день, полное
освоение урока перед переходом к следующему), затем — как стартап с продвижением через
TikTok. Планка качества высокая: каждый урок практичный, интерактивный, со звуком.

Старый общий курс «English from Zero» **переписываем под фермерский формат**. Принятый план:

- **Архитектура:** урок = единый объект `{grammar, words, dialogue, quiz}`; навигация
  Hub-and-Spoke (главная-хаб со списком 15 уроков → экран урока с табами
  Grammar/Words/Dialogue/Quiz). `app/` переписываем с нуля.
- **Данные:** один внешний `app/data.js` → `window.LESSONS = [ … ]` (без `fetch`, offline).
- **Старое в архиве:** `lessons/`, `vocabulary/`, `tracks/`, старые `README/PROGRESS/HOW-TO-STUDY`
  перенесены в `_legacy/` (не удалять — могут пригодиться как черновик грамматики).
- **Сначала лексикон, потом код (Фаза 0):** мастер-лексикон уроков 1–15 (en + IPA из словаря +
  ru + emoji) со сверкой по CEFR-PDF и стартовым whitelist снежка — одобряется владельцем ДО кода.
- **Сборка вертикальным срезом:** затем движок + эталонный Урок 1 по «золотому стандарту»,
  владелец проходит его как первый пользователь, затем тираж уроков 2–15 (по одному в день).
- **MVP-движок:** хаб+прогресс, урок с 4 табами, Web Speech, квиз-тренажёр, streak.
  Сертификат, flashcard-режим, PWA, распознавание речи — позже.
- **Git:** репозиторий инициализирован для обратимости (данные урока живут в JS).

## Архитектура (Zero-Budget, Offline-First)

Без сервера, без БД, без внешних зависимостей (никаких JS/CSS-фреймворков).

- **Стек:** HTML + CSS + vanilla JS. Хостинг — GitHub Pages.
- **Аудио:** Web Speech API, `lang='en-GB'`, `rate=0.88`; всегда `speechSynthesis.cancel()`
  перед новым воспроизведением. Нет интернета → остановить и показать сообщение.
- **Хранение:** localStorage (прогресс по уровню, счётчик слов, streak, сертификат).
  Персональных данных на сервере нет, аутентификации нет.
- **Иконки:** emoji (эмодзи строго по смыслу слова). Картинок-файлов нет.
- **Производительность:** < 3 c на 3G, данные урока inline, работает offline после
  первой загрузки.
- **Совместимость:** Chrome/Samsung Internet/Safari iOS; Firefox Android — частично (Speech API).

## Формат данных урока (RU-only)

Один урок = объект с `grammar / words / dialogue / quiz`. Сейчас только русские поля
(`title_ru`, `rule_ru`, `tr_ru`, `hint_ru`, `ru`), плюс англ. `en`, IPA `pn`, кириллическая
`transcr`. **Localization-ready:** суффикс `_ru` сохраняем намеренно — добавить языки в
будущем = дописать `_uz/_tj/_kg/_kz` рядом, без переименований. UI-строки держим
централизованно (`UI.ru = {…}` + хелпер `t(key)`), чтобы локализация была дописыванием,
а не переписыванием разметки. Сам массив уроков — в `app/data.js`.

- **grammar:** 3 формы (✅ positive / ❌ negative / ❓ question), у каждой — `table`
  (`subj/verb/example/transcr/tr_ru`); `intro` через ❌/✅; `cultural` (различие с
  русским); `note` (типичная ошибка ⚠️). Суммарно **≥10 примеров**. Списки слов/местоимений
  в правилах — CSS-grid (`grid-template-columns:1fr 1fr`), не сплошной строкой.
- **words:** **≥30** уникальных объектов `{e, en, transcr, ru, pn}`. По одному слову на
  элемент (запрещено `en:'one, two, three'`).
- **dialogue:** **≥10** реплик, чередование спикеров `s`: `m`(manager)/`w`(worker)/
  `c`(cashier)/`d`(doctor). Длина реплики 5–15 слов.
- **quiz:** **≥10** заданий `{q, opts[4], c, expl, hint_ru}`. Инструкции — только тегами
  `[COMPLETE]`/`[TRANSLATE]`/`[NEGATIVE]`/`[CORRECT]`/`[QUESTION]` (раскрываются в RU),
  английских инструкций в тексте быть не должно. `expl` — обязательное объяснение ответа.

## Золотой стандарт контента (педагогика — НЕ нарушать)

1. **Снежный ком:** в грамматике/диалогах/тестах урока N можно использовать ТОЛЬКО слова
   из словарей уроков 1…N **плюс стартовый whitelist служебных/грамматических слов**
   (`am/is/are`, `not`, `do`, `a/the`, `this`, `where/what`, местоимения…), вводимых в
   блоке grammar и не входящих в 30 тематических. Без этого whitelist снежок на Уроке 1
   неисполним (учить отрицание/вопрос только «фермерскими существительными» нельзя).
2. **Уникальность словаря:** одно и то же англ. слово появляется в `words` только в одном
   уроке за весь курс (без дублей).
3. **Транскрипция везде:** кириллическая `transcr` обязательна для каждого слова, примера,
   строки таблицы и реплики; в правилах англ. слова оборачивать `<span class="g-transcr">`.
4. **Нулевой уровень:** объяснения простым языком без лингвотерминов; базовые частицы/
   местоимения переводить явно (`I am (я)`).
5. **Только фермерская лексика:** запрещены «городские/офисные»/нерелевантные слова
   (printer, computer, washroom attendant, apple и т.п.). Один урок = одна тема.
6. **Полнота:** грамматика ≥10 примеров, words ≥30, dialogue ≥10, quiz ≥10. Меньше —
   урок не готов.

> Эти правила проверяются Node-скриптами в `scripts/`: `audit.js` (структура/полнота),
> `dedup.js` (уникальность словаря), `snowball.js` (снежный ком). Запуск — **`npm run audit`**.
> Прогонять перед каждым коммитом урока; пустой список ошибок → урок «золотой».

## Контент-валидация и риски (из совета 2026-06-16)

Главный риск проекта: владелец с нулевым английским — и автор контента (через агента), и
приёмщик; аудит-скрипты ловят структуру, но НЕ фонетику/естественность. Поэтому:

- **IPA — из авторитетного словаря** (Wiktionary/CMUdict), не выдумывать. Кириллическая
  `transcr` — производная от IPA, подаётся как **затухающая опора** (больше в ранних
  уроках, меньше дальше), а не вечный костыль: цель — научить читать латиницу (payslip’ы,
  таблички). Слово всегда показывать латиницей + 🔊; кириллица вторична.
- **Мастер-лексикон — артефакт Фазы 0** (см. план): все слова 15 уроков с IPA, сверенные с
  CEFR A0-A1 PDF, ДО кода. Это снимает разом снежок, дедуп, CEFR-объём и галлюцинации IPA.
- **Зона повышенной ответственности:** деньги/зарплата, права, NHS/медицина, безопасность —
  это «лексика выживания»; неверная фраза = реальный вред. Такие категории проверять строже,
  без юридических/медицинских утверждений; табу про визу/заработок сохраняется.
- **CEFR — это области лексики, не жёсткий потолок слов.** «Фермерская лексика» = фермерское
  подмножество CEFR-областей (jobs→picker/packer, food→…, objects→punnet/crate). Объём ≥450
  достижим, но проверить по PDF до тиража.
- **Web Speech офлайн-зависим:** нет голоса/сети → не молчать, а показать сообщение и текст.
- **localStorage хрупок** (чистка кэша/переустановка стирает прогресс) → позже экспорт/импорт.
- **Владелец ≠ репрезентативный пользователь** (грамотнее, мотивированнее аудитории): его
  прохождение проверяет педагогику, но не доступность для A0/низкой цифровой грамотности.

## Дизайн (Telegram Web 2026, light-only)

- **Только светлая тема.** Тёмной нет — не добавлять.
- Clean Light: фон `#f0f2f5`, карточки `#fff`, текст `#1a1a2e`, акцент-синий `#0075de`.
- Системный sans-serif (`-apple-system, 'Segoe UI', Roboto`), без Google Fonts-зависимостей.
- Контейнер `max-width:430px; margin:0 auto` (мобильный приоритет, 320–430px).
- Навигация Hub-and-Spoke: главная-хаб + плавающая кнопка «Назад» (iOS-chevron). Внутри
  урока — табы Grammar/Words/Dialogue/Quiz с кнопками «Далее ➔».
- Карточки `border-radius:12–16px`; чипы pill `99px`; прогресс-бар `transition:width .5s`;
  нажатие `scale(0.98)`. Словарь — сетка 2 колонки (моб.) / 3 (широкий).

## Доступность

- Шрифт ≥14px (контент), ≥11px (метки/транскрипция); line-height ≥1.4; контраст WCAG AA.
- Touch target ≥44×44px, отступ между кнопками ≥8px, видимый отклик на нажатие.
- Не полагаться только на цвет: ✓ зелёный + «Правильно!», ✗ красный + «Неверно».
- Без горизонтального скролла: таблицы переносятся, кнопки 🔊 всегда в зоне видимости.

## Git-безопасность

Данные уроков живут внутри JS-файла → откат через git может стереть невосстановимый
контент. Поэтому: частые коммиты после каждого готового урока; бэкап массива перед
опасными операциями; стадить файлы по имени, не `git add .`. «Откати» = неразрушающий
просмотр (`git checkout <commit>`/`stash`/`log`), не `reset --hard`.

## Чего не делать

- ❌ Не строить СЕЙЧАС переключатель языков и 5-язычные данные (UZ/TJ/KG/KZ). Но архитектуру
  держать localization-ready (`_ru`-суффиксы + централизованные UI-строки) — локализация в планах.
- ❌ Не добавлять тёмную тему.
- ❌ Не вводить внешние зависимости/фреймворки, сервер, БД, аутентификацию.
- ❌ Не использовать в уроке слова вне «снежного кома» и нефермерскую лексику.
- ❌ Не хардкодить пути из чужих спек (`a1.html`, `translations.js`, `scratch/`, `fe_*`)
  как наши — сверяйся с реальной структурой репозитория.
